# RBAC — Модель ролей и разрешений

## Роли

| Роль | Код | Описание |
|------|-----|----------|
| Администратор | `ADMIN` | Полный доступ ко всему. Управление пользователями, справочниками, настройками. |
| Наниматель | `HIRING_MANAGER` | Инициирует вакансии, участвует в собеседованиях, видит свои вакансии и кандидатов. |
| Рекрутер | `RECRUITER` | Основной оператор: поиск, скрининг, проведение этапов, отправка офферов. |
| HR | `HR` | Проверка СБ, онбординг, испытательный срок. Не управляет скринингом/интервью. |

## Ограничение данных по ролям

### HIRING_MANAGER
- Видит только **свои вакансии** (созданные им или назначенные через assignments)
- Видит кандидатов только по **своим вакансиям**
- Не может создавать/редактировать кандидатов
- Не может менять стадии кандидатов
- Не имеет доступа к модулям: Проверка СБ, Онбординг, Испытательный срок

### RECRUITER
- Видит **все вакансии** и **всех кандидатов**
- Может создавать/редактировать кандидатов, менять стадии
- Может создавать и отправлять офферы
- Не может согласовывать офферы (только отправлять на согласование)
- Не имеет доступа к модулям: Проверка СБ, Онбординг, Испытательный срок

### HR
- Видит **все вакансии** и **всех кандидатов** (для чтения)
- Управляет модулями: Проверка СБ, Онбординг, Испытательный срок
- Может комментировать кандидатов
- **Не может** создавать/редактировать вакансии
- **Не может** управлять стадиями кандидатов
- **Не может** создавать/отправлять офферы

### ADMIN
- Полный доступ ко всему
- Единственный, кто управляет: пользователями, справочниками, флоу, видимостью тегов

## Текущие ограничения (заметки для корректировки)

1. **Нет проверки "только свои вакансии" для HIRING_MANAGER** на уровне БД — фильтрация идёт в `vacancies.service.ts` через `assignments`.
2. **HR не ограничен в просмотре** — может видеть все вакансии/кандидатов, но не может их менять.
3. **RECRUITER не ограничен по вакансиям** — видит и редактирует все.
4. **Оффер-согласование** — currently implemented через OfferApproval, но нет явной проверки что approver — это HIRING_MANAGER.
5. **Теги** — видимость фильтруется на уровне API для не-ADMIN, но фильтрация не применяется при просмотре кандидатов/вакансий с тегами.
